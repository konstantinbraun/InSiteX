2     Login

InSite 3 kann mit einem Browser jeweils nur einmal aufgerufen werden. Es gelten jeweils die Parameter (Benutzer, Anwahl Bauvorhaben, usw.) des letzten Aufrufs.

Sollen von einem Arbeitsplatz gleichzeitig mehrere Aufrufe erfolgen, so müssen jeweils unterschiedliche Browser (Chrome, Firefox, usw.) benutzt werden.

Die Login-Maske ermöglicht folgende Varianten des Programmaufrufs:

 

 

2.1       Anmeldung

Für die normale Benutzeranmeldung gelten folgende Voraussetzungen:

►    Für den Benutzer muss ein Stammdatensatz angelegt sein.

►    Dieser Stammdatensatz muss freigegeben sein.

►    Dem Benutzer muss mindestens ein Bauvorhaben und eine Rolle in diesem Bauvorhaben zugeordnet sein.

Ist dem Benutzer noch kein Bauvorhaben zugeordnet, fährt die Verarbeitung automatisch gem. 2.2.2 Firma an Bauvorhaben anmelden fort.

2.2       Registrierung

Die Registrierfunktionen ermöglichen es, dass initiale Daten durch Personen, die noch nicht als Benutzer im System (Benutzerstamm) hinterlegt sind, erfasst werden können.

Folgende Fälle werden unterstützt:

 

 

2.2.1  Firma für zentralen Firmenstamm registrieren

Firmen, die bisher nicht in InSite 3 existieren, werden quasi per „do-it-yourself“ von qualifizierten Mitarbeitern dieser Firmen mit ihren Stammdaten im System hinterlegt.

Bei diesem Vorgang müssen auch die Stammdaten für einen „Firmen-Administrator“ erfasst werden. Dieser ist zukünftig berechtigt alle firmenbezogenen Daten (insbesondere Mitarbeiter-Stammdaten) zu pflegen.

Weitere Details siehe Dokument „Firmenstamm“.

 

 

2.2.2  Firma an Bauvorhaben anmelden

Unter der Voraussetzung, dass die Firma für den zentralen Firmenstamm und ihr zugehöriger Firmen-Administrator im zentralen Benutzerstamm freigegeben wurden (durch den System-Administrator), kann der Firmen-Administrator hierüber beantragen, dass seine Firma für ein bestimmtes Bauvorhaben freigegeben wird.

Zu diesem Zweck muss der Benutzer folgende Felder eingeben:

►   

►    Mit ruft er folgenden Dialog auf:

Eingegeben wird:

►    Bauvorhaben

Die Auswahlliste führt alle Bauvorhaben auf, bei denen die Firma noch nicht eingetragen ist.

►    Auftraggeber

Die Firma ist in dem Bauvorhaben Subunternehmer dieses Auftraggebers. In der Auswahlliste werden alle freigegebenen Firmen des Bauvorhabens aufgeführt.

 

 

2.2.3  Mitarbeiter für Bauvorhaben registrieren

Mit dieser Funktion wird die Erfassung der Mitarbeiterstammdaten auf den einzelnen Mitarbeiter delegiert. Zu diesem Zweck generiert der Firmen-Administrator per Firmenstamm einen eindeutigen Anmeldecode, den er an die Mitarbeiter weitergibt.

Per Eingabe

►    des Anmeldecodes und

►   

ruft der Mitarbeiter die Eingabemaske für seine Daten auf.

Durch den Anmeldecode wird zum einen der Missbrauch ausgeschlossen und es erfolgt die Zuordnung der Mitarbeiterstammdaten zur jeweiligen Firma.

 

 

3     Benutzer-Menü

 

 

3.1       Home

Mit diesem Button positioniert die Anwendung auf die Startseite von InSite 3.

 

 

3.2       Benutzer

Per Klick auf den Button „Benutzer“ öffnet ein Untermenü, mit dem folgende Funktionen aufgerufen werden können:

►    Sprache ändern

Der Benutzer wählt für die InSite 3 Dialoge eine abweichende Sprach-Darstellung

►    Kennwort ändern

Der Benutzer ändert sein persönliches Kennwort.

►    Auswahl Bauvorhaben

Der Benutzer wechselt zu einem anderen Bauvorhaben. Er kann aus einer Liste der Bauvorhaben, für die er zugelassen ist, auswählen. 

►    Abmelden

Der Benutzer meldet sich von der InSite 3 Anwendung ab.

 

 

3.3       Ansicht

Per Klick auf den Button „Ansicht“ öffnet ein Untermenü, mit dem folgende Funktionen aufgerufen werden können:

►    Aktualisieren

Der Benutzer aktualisiert die angezeigte Seite.

►    Layout ändern

Der Benutzer kann das Layout der Maskendarstellung in InSite 3 anpassen.

Empfohlen wird das Layout „Windows7“.

 

 

3.4       Administration

►    LogViewer

Ermöglicht die Analyse der Benutzeraktivitäten in InSite 3 bezogen auf die jeweiligen Verarbeitungen.

4     Anwendungsmenü

Die Anzeige des Anwendungsmenüs erfolgt benutzerbezogen in Abhängigkeit von:

►    Der Rolle des jeweiligen Benutzers.

D.h. Menüknoten und Funktionen, die für eine Rolle nicht freigeschaltet sind, werden nicht angezeigt.

►    Der persönlichen Historie des Benutzers.

D.h. es wird das Menü in derselben Detaillierung dargestellt wie bei der letzten Session des Benutzers.

Die Bedienung des Menüs ist wie folgt:

►    Per Klick auf eine Kapitelüberschrift oder das führende Symbol (Darstellung ist abhängig vom Layout) wird der jeweilige Menüknoten erweitert (aufgeklappt) oder reduziert.

►    Per Klick auf die Bezeichnung eines Menüpunktes wird die korrespondierende Verarbeitung aufgerufen.

5.2       Pflegemasken

Die Pflegemasken haben folgende Systematik:

►    In Feldern, deren gültige Belegung sich aus einer Auflistung (Drop Down) ergibt, sollten die ersten Buchstaben des gewünschten Elementes eingegeben werden. Hiermit werden die möglichen Elemente, aus denen ausgewählt werden kann, stark reduziert.

►    Felder, für die eine Zwangseingabe programmiert, bzw. konfiguriert ist, werden mit einem fetten Führungstext angezeigt.

►    Wenn die Menge der Daten eine Aufteilung in mehrere Teilmasken erfordert, werden diese über die Kopfleiste in Form von Registern aufgerufen.

►    Eine Veränderung muss über die Tasten abgeschlossen werden.

►    Eine Neuanlage muss über die Tasten
abgeschlossen werden.

►    Mit <Enter> wird die Funktionalität gem.  bzw.  ausgelöst.

►    Mit <Esc> wird die Funktionalität gem.  ausgelöst.

►    Mit  bzw. <Enter> wird die Fehlerprüfung ausgelöst.

Erst wenn alle Fehler behoben sind, kann der Datensatz gespeichert werden.

Per  werden die Informationen des Unterdialogs bereits in die Datenbank geschrieben. d.h. selbst wenn der übergeordnete Dialog mit  verlassen wird, sind die Informationen gespeichert.
